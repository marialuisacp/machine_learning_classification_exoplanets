Preparação e leitura dos dados¶

Neste trabalho estamos interessados em resolver um problema de classificação binária de candidatos a exoplanetas encontrados pela sonda espacial Kepler da NASA. Um exoplaneta é um planeta fora do sistema solar (i.e. que não orbita o sol). A sonda primeiro identifica sinais de possíveis exoplanetas, chamados de Kepler Object of Interest (KOI). Porém nem todos os KOIs são de fato exoplanetas, alguns se tratam de falsos positivos de origens diversas. A tarefa é então classificar os KOIs entre exoplanetas confirmados e falsos positivos. Cada linha do arquivo corresponde a um KOI e as features são características estimadas de cada (possível) exoplaneta (tamanho, temperatura, features da estrela hospedeira, etc).

Dados retirados do NASA Exoplanet Archive.

def showVariationsGraph(execs, nameParam):

    plt.ylabel('Acurácia')
    plt.xlabel(nameParam)
    plt.title(nameParam + ' x Acurácia')

Decision Tree¶

Árvores de decisão são métodos de aprendizado de máquinas supervisionado não-paramétricos, muito utilizados em tarefas de classificação e regressão. Este método representa funções como árvores de decisão. Estas árvores são treinadas de acordo com um conjunto de treino (exemplos previamente classificados) e posteriormente, outros exemplos são classificados de acordo com essa mesma árvore.

In [11]:
#

#

def test_decision_tree(max_depth):
    # Instancializa o classificador
    if(max_depth):
        clf = DecisionTreeClassifier(max_depth=max_depth)
    else:
        clf = DecisionTreeClassifier()

    # Treina o classificador
    clf = clf.fit(x_train, y_train)

    # Realiza predição
    y_pred = clf.predict(x_test)
    
    train_acc = clf.score(x_train, y_train)
    val_acc = clf.score(x_test, y_test)

    print("Acuracia do treino: {:.3}".format(train_acc))
    print("Acuracia do teste: {:.3}".format(val_acc))
    return [max_depth, train_acc, val_acc];

O parâmetro max_depth define a altura máxima da árvore. O método foi executado assumindo diferentes valores do parâmetro a fim de enconrar o melhor resultado.

Support Vector Machine (SVM)¶

Uma máquina de vetores de suporte é um método de aprendizado supervisionado que analisa os dados e reconhece padrões, usado para classificação e análise de regressão. O SVM toma como entrada um conjunto de dados e prediz, para cada entrada dada, qual de duas possíveis classes a entrada faz parte, o que faz do SVM um classificador linear binário não probabilístico.

Um modelo SVM é uma representação de exemplos como pontos no espaço, mapeados de maneira que os exemplos de cada categoria sejam divididos por um espaço claro que seja tão amplo quanto possível. Os novos exemplos são então mapeados no mesmo espaço e preditos como pertencentes a uma categoria baseados em qual o lado do espaço eles são colocados.

In [19]:
#
# Funcao que treina o modelo e mostra os resultados utilizando o SVM
#

def test_svm(c, kernel, gamma):
    clf = svm.SVC(C=c, kernel=kernel, gamma=gamma)

    clf = clf.fit(x_train, y_train)

    train_acc = clf.score(x_train, y_train)
    val_acc = clf.score(x_test, y_test)

    print("Acuracia do treino: {:.3}".format(train_acc))
    print("Acuracia do teste: {:.3}".format(val_acc))

O SVM recebe um parâmetro C, que é um parâmetro de otimização. Ele informa à otimização do SVM quanto se deseja evitar classificar erroneamente cada exemplo de treinamento. Para valores grandes de C, a otimização escolherá um hiperplano de margem menor se esse hiperplano fizer um trabalho melhor ao classificar todos os pontos de treinamento corretamente. O valor muito pequeno de C faz com que o otimizador procure por um hiperplano de separação de margem maior, mesmo se esse hiperplano classificar incorretamente mais pontos.

Kernels¶

O SVM introduz a função kernel que transforma o espaço de dados original em um novo espaço com dimensões mais altas. Existem várias fórmulas para a função kernel, neste trabalho estamos trabalhando com o kernel linear, sigmoide, polinomial e radial.

Linear¶

É útil ao lidar com grandes vetores de dados esparsos. É frequentemente usado na categorização de texto. O kernel linear também funciona bem em problemas de regressão. A equação dada por:

$$ K(X,Y)=X^TY $$

Sigmoid¶

Este kernel (no português sigmoide) pode ser usado como proxy para redes neurais. A equação é dada por:

$$ K(X,Y)=tanh(γ⋅X^TY+r) $$

Polinomial¶

Este kernel induz o espaço de combinações polinomiais até certo ponto. Consequentemente, pode-se trabalhar com limites de decisão ligeiramente "dobrados", como parábolas com grau = 2. Em alguns casos combinações polinomiais não são flexíveis o suficiente para "dobrar" o espaço com força suficiente para capturar o ponto de interesse. Este kernel é popular no processamento de imagens. A equação é dada por:

$$ K(X,Y)=(γ⋅X^TY+r)d,γ>0 $$

Radial basis function (RBF)¶

O tipo mais usado de função do kernel é o RBF. Porque tem resposta localizada e finita ao longo de todo o eixo x. É um kernel de propósito geral, usado quando não há conhecimento prévio sobre os dados. A equação é dada por:

$$ K(X,Y)=exp(∥X−Y∥^2/2σ^2) $$

k-Nearest Neighbors (KNN)¶

K-Nearest Neighbors é um dos muitos algoritmos de aprendizagem supervisionada usado no campo de data mining e machine learning, ele é um classificador onde o aprendizado é baseado no quão similar é um dado (um vetor) do outro. O treinamento é formado por vetores de n dimensões.

In [24]:
#
# Funcao que treina o modelo e mostra os resultados utilizando o KNN
#

def test_knn(k):
    clf = neighbors.KNeighborsClassifier(n_neighbors=k)

    clf = clf.fit(x_train, y_train)

    train_acc = clf.score(x_train, y_train)
    val_acc = clf.score(x_test, y_test)

    print("K = {}".format(k))
    print("Acuracia do treino: {:.3}".format(train_acc))
    print("Acuracia do teste: {:.3}".format(val_acc))
    return [k, train_acc, val_acc];

O parâmetro k informa a quantidade de vizinhos. Na fase de classificação, k é uma constante definida pelo usuário e um vetor não rotulado é classificado atribuindo-se o rótulo que é mais frequente entre as k amostras de treinamento mais próximas a esse ponto de consulta.

Random Forest¶

Floresta Aleatória é um algoritmo de aprendizagem supervisionada que cria uma floresta de um modo aleatório. A floresta criada é uma combinação (ensemble) de árvores de decisão, em vários casos treinados com o método de bagging.

In [31]:
#
# Funcao que treina o modelo e mostra os resultados utilizando o Random Forest
#

def test_random_forest(N):
    clf = ensemble.RandomForestClassifier(n_estimators=N)

    clf = clf.fit(x_train, y_train)

    train_acc = clf.score(x_train, y_train)
    val_acc = clf.score(x_test, y_test)

    print("N = {}".format(N))
    print("Acuracia do treino: {:.3}".format(train_acc))
    print("Acuracia do teste: {:.3}".format(val_acc))
    return [N, train_acc, val_acc];

O parâmetro N define a quantidade de árvores que serão criadas pelo Random Forest. O hiperparâmetro N que indica o número de árvores construídas pelo algoritmo antes de tomar uma votação ou fazer uma média de predições. Em geral, uma quantidade elevada de árvores aumenta a performance e torna as predições mais estáveis, mas também torna a computação mais lenta.

Gradient Tree Boosting¶

Gradient Tree Boosting é uma técnica de aprendizado de máquina para problemas de regressão e classificação, que produz um modelo de previsão na forma de um conjunto de modelos de previsão fracos, a partir de árvores de decisão. Ele constrói o modelo em um modo de estágio, como outros métodos de otimização, e os generaliza ao permitir a otimização de uma função de perda arbitrariamente diferenciável.

Avaliação dos métodos¶

Para avaliar os métodos foi escrita a função evaluation_method que recebe o classificador a ser avaliado. Todos os métodos contaram com os melhores valores encontrados na etapa anterior. A avaliação contou com a acurácia e a validação cruzada k-fold com k igual a 5. A função abaixo exibe os valores da acurácia, a curva de ROC a partir da validação cruzada k-fold, a precisão e revocação dos métodos. Além disso a função salva os valores em uma lista para que no final os valores encontrados sejam comparados.

# Função de avaliação de métodos, recebe o classificador a ser avaliado e o nome do método
#

def evaluation_method(clf, nameMethod):
    # Validação cruzada utilizando k-fold
    cv = KFold(n_splits=5)

    # Define variáveis 
    tprs = []
    aucs = []
    mean_fpr = np.linspace(0, 1, 100)
    i = 0
    plt.figure(figsize=(8, 8))

    # Calcula as probabilidades para cada conjunto definido pelo k-fold
    for train, test in cv.split(x, y):
        clff = clf.fit(x[train], y[train])
        y_pred = clff.predict(x[test])
        y_prob = clff.predict_proba(x[test])[:, 1]
                
        # Computa a curva ROC e a área de curvatura
        fpr, tpr, thresholds = roc_curve(y[test], y_prob)

        tprs.append(interp(mean_fpr, fpr, tpr))
        tprs[-1][0] = 0.0
        roc_auc = auc(fpr, tpr)
        aucs.append(roc_auc)
        plt.plot(fpr, tpr, lw=1, alpha=0.3,
                 label='ROC fold %d (AUC = %0.2f)' % (i, roc_auc))
        i += 1
    
    y_pred = clff.predict(x_test)
    precisionMethod = precision_score(y_test, y_pred)
    recallMethod = recall_score(y_test, y_pred)

    print("")
    print("\tPrecision: %1.3f" % precisionMethod)
    print("\tRecall: %1.3f" % recallMethod)
    
    namesMethods.append(nameMethod)
    valuesMethods.append([precisionMethod, recallMethod])
    matrixMethods.append([nameMethod, precisionMethod, recallMethod])

    plt.plot([0, 1], [0, 1], linestyle='--', lw=2, color='r',
             label='Chance', alpha=.8)

    mean_tpr = np.mean(tprs, axis=0)
    mean_tpr[-1] = 1.0
    mean_auc = auc(mean_fpr, mean_tpr)
    std_auc = np.std(aucs)
    plt.plot(mean_fpr, mean_tpr, color='b',
             label=r'Mean ROC (AUC = %0.2f $\pm$ %0.2f)' % (mean_auc, std_auc),
             lw=2, alpha=.8)

    std_tpr = np.std(tprs, axis=0)
    tprs_upper = np.minimum(mean_tpr + std_tpr, 1)
    tprs_lower = np.maximum(mean_tpr - std_tpr, 0)
    plt.fill_between(mean_fpr, tprs_lower, tprs_upper, color='grey', alpha=.2,
                     label=r'$\pm$ 1 std. dev.')

    plt.xlim([-0.05, 1.05])
    plt.ylim([-0.05, 1.05])
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('Curva ROC (' + nameMethod + ')')
    plt.legend(loc="lower right")
    plt.show()

A analise ROC (Receiver Operating Characteristic) é uma ferramenta poderosa para medir e especificar problemas no desempenho de algoritmos.

Na avaliação dos métodos estudados, os resultados de precisão e revocação também foram altos. A árvore de decisão obteve o maior recall e o Random Forest o melhor precision. Em relação aos dois valores: preciosion e recall, acredito que o Random Forest obteve melhores resultados, seguidos pelo Gradient Tree Bosting. O Gradient Tree Bosting, quando executado com 500 iterações obteve o melhor resultado de precisão e revocação, no entanto a execução foi lenta o que fez com que fosse escolhido um bom número de iterações 100.

Em geral todos os modelos apontaram para um alto valor de acertos (True Positives), talvez pela quantidade de dados ser pequena. Os resultados de todos os métodos testados foram considerados muito bons.

Outro ponto importante é que podemos observar uma curva similar em todos os métodos. Ou seja, pela parametrização e experimentação, foi possível obter valores altos, talvez pelo tamanho pequeno dos dados.
